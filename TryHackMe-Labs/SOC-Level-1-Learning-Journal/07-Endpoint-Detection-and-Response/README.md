# Endpoint Detection and Response (EDR)

## Overview

Completed a hands-on Endpoint Detection and Response (EDR) lab focused on understanding security telemetry, endpoint visibility, detection logic, and threat indicators within an EDR platform.

The room emphasized how security analysts leverage EDR solutions to investigate suspicious activity, review endpoint events, analyze process execution chains, and identify potential threats through available telemetry and detection data.

> Note: Incident acknowledgement, containment, and response actions were outside the scope of this room. The primary focus was understanding the visibility and investigative capabilities provided by EDR technologies.

## Detection Scenarios Reviewed

1. Initial Access via Malicious Office Document
2. Suspicious Persistence via Scheduled Task
3. Unsigned Binary Network Activity
4. Credential Dumping via LSASS Memory Access

## Key Findings

* Examined process execution chains and parent-child relationships.
* Identified malicious use of cURL for payload delivery.
* Investigated scheduled task persistence mechanisms.
* Analyzed suspicious outbound network activity.
* Reviewed credential dumping activity targeting LSASS.
* Validated indicators using threat intelligence context.
* Mapped malicious behaviors to MITRE ATT&CK techniques.
* Reviewed EDR detection metadata, severity ratings, and confidence scores.
