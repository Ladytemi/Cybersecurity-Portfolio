# Introduction to SIEM

## Overview

Completed the Introduction to SIEM room and gained hands-on experience using a Security Information and Event Management (SIEM) platform to investigate security alerts and identify suspicious activity.

## Skills Demonstrated

- Security Information and Event Management (SIEM)
- Alert Investigation
- Event Log Analysis
- Rule Analysis
- Threat Detection
- Security Monitoring
- Incident Triage
- True Positive / False Positive Validation
- Host and User Attribution
- Security Operations Center (SOC) Workflow

## Investigation Summary

The lab involved reviewing SIEM dashboards, examining event logs, analysing detection rules, and validating security alerts.

The investigation focused on:

- Identifying the process responsible for triggering an alert
- Determining the user associated with the suspicious activity
- Identifying the affected hostname
- Reviewing the detection rule logic
- Determining which keyword triggered the alert
- Classifying the alert as a True Positive or False Positive
- Applying the appropriate response action

## Key Findings

- Identified a suspicious cryptomining process execution.
- Investigated the event responsible for triggering the alert.
- Determined the associated user and host involved.
- Reviewed SIEM detection rules and matching conditions.
- Classified the activity as a True Positive.
- Selected the appropriate containment response.

## Outcome

Successfully completed the Introduction to SIEM room and demonstrated the ability to investigate alerts, analyse SIEM data, validate detections, and support incident response decisions.
