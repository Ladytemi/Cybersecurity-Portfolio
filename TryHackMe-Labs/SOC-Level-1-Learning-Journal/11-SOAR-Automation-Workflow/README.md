# SOAR Automation Workflow

## Room
TryHackMe - SOAR

## Objective
Learn how Security Orchestration, Automation and Response (SOAR) platforms automate security operations by integrating threat intelligence, incident response, reputation checks, data extraction and case management workflows.

## Skills Practiced

- SOAR fundamentals
- Security automation concepts
- Threat intelligence integration
- Incident response workflow automation
- Reputation checking
- Case management automation
- Course of Action (COA) orchestration
- SOC workflow optimization

## Activities Performed

### Threat Intelligence Feeds
Configured automated retrieval of threat intelligence data and alert ingestion workflows.

### Incident Data Extraction
Automated extraction of:

- Domains
- URLs
- IP Addresses

while maintaining analyst oversight for unknown threats.

### Reputation Checks
Integrated automated reputation validation and sandbox testing processes.

### Course of Action (COA)
Configured automated response actions including:

- Blocking malicious domains
- Blocking malicious URLs
- Blocking malicious IP addresses
- Updating case tickets

while preserving analyst approval before remediation.

### Case Management
Automated:

- Case creation
- Case assignment
- Ticket communication
- Ticket updates

while retaining manual control for ticket deletion.

## Key Takeaways

- SOAR reduces repetitive analyst workload.
- Automation improves response speed and consistency.
- Human validation remains important for high-risk decisions.
- Effective SOAR implementation combines automation with analyst oversight.
