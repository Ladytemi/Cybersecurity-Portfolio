# Deloitte Suspicious Activity Analysis

## Simulation

Deloitte Australia Cyber Job Simulation - Forage

## Task

Task 1: Cyber Security

## Overview

This project involved reviewing web request logs for Daikibo Industrials after a suspected breach of their internal telemetry status dashboard.

The objective was to determine whether the alleged breach could have occurred directly from the internet without VPN access and to identify suspicious automated activity within the provided `web_requests.log` file.

## Background

Daikibo Industrials experienced a production issue that disrupted assembly lines and affected supply chain operations. A major news publication later revealed sensitive internal information, raising concerns that the company's telemetry dashboard may have been accessed improperly.

## What I Did

* Reviewed structured web request logs from Daikibo's internal network.
* Analyzed request patterns across multiple internal IP addresses.
* Looked for unusual access sequences involving login activity, dashboard resources, and API calls.
* Identified signs of repeated automated API requests.
* Assessed whether suspicious activity could have originated externally without VPN access.
* Used log analysis techniques to identify the user linked to suspicious activity.
